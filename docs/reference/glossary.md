# Glossary

- Session workspace: frontend state and operations for the selected project and session.
- Pi session: persisted pi conversation log.
- Runtime session: active pi runtime handle for executing work.
- Transcript detail: selected session branch projected for display.
- Workbench: main selected-session area.
- Session handoff: state transition around creating or selecting a session.
- Startup run: first session creation and prompt flow from the start screen.
- Runtime controls: model, thinking, context, tools, stop, reload, and related controls.
- Tool row: collapsed or expanded transcript row for tool activity.
- Thinking trace: model reasoning output shown when available.
- Compaction: pi context compaction flow.
- Fork: new session created from an existing entry.
- Pierre preview: inline or fullscreen file, image, patch, or diff preview.
