# Runtime controls

Runtime controls include the model picker, thinking picker, context usage popover, and tool count popover, including `0 tools`.

For an existing session, controls apply to the selected runtime session. On the start screen, model and thinking choices are staged before a session exists and applied during startup.

The backend uses `one-at-a-time` steering and follow-up mode. Leyline does not expose a UI control for changing that mode.
